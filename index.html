Жизненный цикл программы

Software Development Life Cycle (SDLC)

последовательность этапов, которые проходит программное обеспечение с момента идеи до его завершения, включая проектирование, разработку, тестирование, внедрение и сопровождение

Этапы разработки

  • Анализ требований - что именно должно делать ПО
  • Проектирование - создаёт структуру и архитектуру системы
  • Разработка - обеспечивает реализацию требований
  • Тестирование - гарантирует, что программа будет работать корректно
  • Сопровождения - обеспечивается долговечность и актуальность программного обеспечения

Модели жизненного цикла разработки

Водопадная модель

Waterfall

каждый этап выполняется один за другим

Спиральная модель

Spiral

одни и те же этапы, но с постепенным уточнением требований и улучшением продукта

V-модель

каждый этап разработки связан с соответствующим этапом тестирования

Agile

Scrum, Kanban

Этапы разработки проходят параллельно в нескольких коротких итерациях (спринтах)

Выбор подходящей модели жизненного цикла

Характер проекта

Сложные - waterfall
Динамичные - agile

Риски

Спиральная для неопределенности и рисков

Размер команды

Для небольших команд - agile

Критичность ПО

Высоконадежное ПО - V-модель

Программный продукт

комплекс программных средств, созданных для решения конкретных задач пользователей

ПО готовое для обычных пользователей

Этапы до ПП

Прототип

первоначальная версия программы, созданная для демонстрации основных функций и интерфейса

Демонстрационная версия

ограниченная по функциональности или времени использования версия программного продукта, предназначенная для ознакомления пользователей с программой перед покупкой

Жизненный цикл программного продукта

  1. Создание - разработка программного продукта, анализа требований и внедрение
  2. Создание - использование продукта, его сопровождение, обновление и поддержка
  3. Создание - исправление ошибок, новые функции и поддержание совместимости
  4. Создание - заменен новым решением, если требования изменились и устарело

Характеристики ПО

  • Функциональность
  • Надежность
  • Эффективность
  • Безопасность
  • Удобство использования
  • Совместимость
  • Поддерживаемость и сопровождаемость
